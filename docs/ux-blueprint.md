# UX Blueprint

## User flow

Landing

↓

Upload screenshot

↓

AI analysis loading state

↓

Report page

## Page inventory

### Landing

- Product promise
- Who it is for
- How the workflow operates
- Preview of report structure
- Primary CTA into upload

### Upload

- Screenshot picker
- File validation
- Preview before submission
- Submit action

### Loading

- Progress headline
- Rotating analysis states
- Reassurance that the system is checking multiple UX dimensions

### Report

- Overall summary
- Section navigation
- Grouped issue cards
- Suggested improvements

## Report information architecture

The report should be organized into four core sections:

- Visual hierarchy
- Accessibility
- Interaction clarity
- Layout issues

Each issue card should include:

- A short title
- A concise explanation
- A recommendation
- Severity or impact signal

## Key interaction principle

The loading state is part of the product, not a gap between pages. It should explain what is happening so the user understands the AI is performing structured review rather than stalling.

## Wireframe intent

The MVP will focus on four screens:

- Landing
- Upload
- Loading
- Report
